# QuoteForge Design Principles

These principles guide every product decision.

> Every decision should help a tradesperson finish today's paperwork faster than they could without our software.

---

## Help, Don't Hinder

Every screen should make life easier.

Nothing should create unnecessary work.

---

## AI Assists. Humans Decide.

AI is a helper.

It is never the product.

The tradesperson remains in control.

---

## Save Time

Every feature should reduce administration.

If it doesn't save time, question why it exists.

---

## Reduce Mental Load

Users should never wonder:

"What do I do next?"

The software should guide them.

---

## Stay Organised

Everything should have a place.

Customers.

Proposals.

Jobs.

Documents.

Notes.

Nothing gets lost.

---

## Never Type It Twice

If information already exists, reuse it.

Never ask users for the same information repeatedly.

---

## Professional By Default

Every proposal should look professional before the user edits anything.

---

## Simple Beats Clever

Choose the simpler solution whenever possible.

---

## Finish The Day

The goal of QuoteForge is simple.

A tradesperson should be able to finish their paperwork quickly, close the laptop, and enjoy the rest of their evening.

---

## Every Screen Has One Job

Every page answers **one simple question**. Build around that question — not around a form.

| Screen | Question |
|--------|----------|
| Home | What do I need to do today? |
| Customer | What do I know about this customer? |
| Proposal | What still needs doing? |
| Job | Am I ready to start? |

Landing page → Build trust.

Settings → Configure once.

### Actions, not forms

Never design around forms. Design around **actions**.

The user describes the work. The software organises everything else in the background wherever possible.

Every screen should feel calm, obvious, and fast.

# Product Principles

## 1. Mobile First

QuoteForge is a phone-first web application.

Do not adapt desktop layouts. Redesign every screen from first principles for mobile use.

Every feature must work beautifully on a mobile phone.

If it doesn't work on a phone, it isn't finished.

The phone experience is the primary product. Desktop and tablet expand from mobile — never the reverse.

Design every interaction around speed. Sessions are 5–10 minutes, often one-handed.

---

## 2. The Software Does The Thinking

The user provides the information.

The software organises it.

The software writes it.

The software formats it.

The software keeps everything organised.

The tradesperson should never have to think about where information belongs.

---

## 3. AI Should Feel Invisible

Users should never need to understand AI.

They simply describe the work.

The software does the rest.

AI exists to reduce thinking, not create more of it.

---

## 4. Less Typing

Collect information once.

Reuse it everywhere.

Never ask the user for the same information twice.

---

## 5. Simple Beats Clever

Always choose the simplest solution.

Avoid unnecessary options.

Avoid technical language.

Avoid complicated workflows.

---

## 6. Speak Like A Tradesperson

Use clear, everyday language.

Never sound corporate.

Never sound technical.

If a builder wouldn't naturally say it, don't use it.

---

## 7. Save Time

Every feature must answer one question:

"Does this save the user time?"

If not, it doesn't belong.

---

## 8. The Kitchen Table Test

Imagine someone has just finished a long day on the tools.

They're tired.

They're sitting at the kitchen table.

They want to finish today's paperwork in minutes.

If a feature feels complicated in that situation,

it needs redesigning.

## Capture First

If a user thinks of something while they're on the move, they should be able to capture it in seconds.

The software should organise it automatically.

The user should never worry about forgetting important information.

## Reduce Mental Load

Our users already make hundreds of decisions every day.

The software should remove decisions, not create them.

Users should never have to remember:

- what needs ordering
- what still needs sending
- what materials are missing
- which jobs are ready
- which customers are waiting

The software remembers.

The user works.

## Information Has One Home

Every piece of information should only need to be entered once.

From that single piece of information, the software should automatically update every relevant part of the system.

Users should never duplicate work.

The software should connect everything together.

## Speak Like The User

The interface should use the words our customers naturally use.

Prefer familiar language over technical or business terminology.

If a tradesperson says "quote" more often than "proposal", use "Quote".

If they say "job" instead of "project", use "Job".

The software should feel familiar from the first time it is opened.
