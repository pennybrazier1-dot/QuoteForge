# Feature Principles

Before adding any feature, ask:

1. Does this save the tradesperson time?

2. Does this help them win more work?

3. Does this make the proposal more professional?

4. Will most tradespeople use this every week?

5. Can this be achieved without adding unnecessary complexity?

If the answer to most of these is "No", reconsider whether the feature belongs in QuoteForge.

Every feature we build should:

- Save time.
- Reduce paperwork.
- Reduce typing.
- Reduce decisions.
- Work perfectly on mobile.
- Be usable without training.
- Feel obvious.
- Reuse existing information.
- Follow the existing design system.
- Keep the interface clean.
- Avoid unnecessary steps.
- Make the user's evening easier.

Every feature should leave the user with less work than before they opened the app.

Our mission: **Finish work. Not paperwork.**

## Phone-first

QuoteForge is a phone-first web application.

Do **not** adapt desktop layouts. Redesign every screen from first principles for mobile.

The mobile experience is not a smaller desktop experience.

Desktop and tablet expand naturally from the mobile design.

Never shrink desktop layouts to fit a phone.

### Who we are designing for

Independent tradespeople with 5–10 minutes, often one-handed — beside the van, between jobs, in the vehicle, or at the kitchen table after work.

### What every screen must do

- Answer one simple question (see screen purposes below)
- Lead with actions, not forms
- Let the user finish one clear job quickly
- Keep the most important action obvious
- Minimise typing and decisions
- Organise information automatically in the background
- Work without training
- Feel calm, obvious, and fast

### One question per screen

| Screen | Question |
|--------|----------|
| Home | What do I need to do today? |
| Customer | What do I know about this customer? |
| Proposal | What still needs doing? |
| Job | Am I ready to start? |
| Money | How is my business doing financially? |
